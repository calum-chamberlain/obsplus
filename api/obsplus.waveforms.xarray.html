obsplus.waveforms.xarray.aggregate.aggregate(dar, method, level, dim='seed_id', coord=None, **kwargs)[source]¶

Apply a channel aggregation over a data array using dim.

The dimension referenced by the variable dim must be a seed id str which has the following format: network.station.location.channel, e.g. BW.RJOB..EHZ

Parameters:
  • dar (DataArray) – The data array to aggregate
  • method (Union[str, Callable]) – The method for channel aggregation, can either be a str which refers to a method in detex.utils.aggregate_methods dict, a callable, which will be passed to the xr.GroupBy.apply method, or a method supported by the xr.GroupBy class, like “mean” and “std”
  • level (Optional[str]) –

    The seed_id level over which to apply the aggregation. Supported options are:

    ’all’ - aggregate all channels in a waveforms together ‘network’ - Multiplex all channels with a common network together ‘station’ - Multiplex all channels with a common station together
  • dim (str) – The dimension over which to aggregate. Will almost always be seed_id.
  • coord (Optional[str]) – A coordinate that has the dim (seed_id by default) as a subset that should also be truncated to reflect the aggregation. If None dont attempt to reconcile any coords with the aggregation level. For example, if a coordinate had an entry “subspace_1_UU.HDR..BHZ” and the aggregation was performed on station, it would be changed in place to “subspace_1_UU.HDR”.
Returns:

The aggregated array, with the aggregated dimension renamed to the value of the variable dim
Return type:

xr.DataArray

obsplus.waveforms.xarray.utils.sel_sid(dar, seed_id)[source]¶

Slice a detex Data Array based on a seed_id.

Seed id normally has the form: network.station.locaton.channel and allows for unix style search strings. Lower level ids can be excluded for implicit wildcards.

Parameters:
  • dar (DataArray) – Data array, generated by :function: ~detex.utils.waveform2array
  • seed_id – The str specifying the seed id. Allows truncations and wildcards.

Notes

the following are all valid values for seed_id:
BW - select all data with BW as network BW.RJOB - select all data from station RJOB on network BW BW.RJOB..EHZ - select only data from this channel BW.R* - select all station in BW that start with an R BW.???B - select all station in BW with 4 chars that end with B
Returns:A DataArray containing the desired seed_ids
Return type:xr.DataArray

obsplus.waveforms.xarray.utils.trim_waveform_array(dar, trim, is_timestamp=True, remove_nan=True, aggregate_by=None, aggregate_func=<function nanmean>)[source]¶

Trim or extend the waveform array either independently for each waveform or holding relative times between any waveforms constant.

Can use the name of an existing coordinate (eg ‘p_times’, ‘origin_time’), a data array that shares some dimensions with the dar, or number which can refer to a time stamp or a time relative to the start of the array.

Parameters:
  • dar (DataArray) – The Data Array with waveform vectors
  • trim (Union[str, DataArray, float]) – The name of a coordinate (all float values) for trimming, a single float, or another data array (which shares some dims with dar) that contains either timestamps or relative trim times.
  • is_timestamp (bool) – If True, the values given by trim are timestamps (ie seconds from 1970). If False they are relative to the traces start time.
  • remove_nan (bool) – If True remove the NaN values from the end of the arrays
  • aggregate_by (Optional[str]) – If not None, must be the name of a coord in dar (eg “group”) or a value supported by the function obsplus.waveforms.xarray.ggregate.
  • aggregate_func (Callable) – Any function that takes a numpy array and returns a single value. Since NaNs are possible it is wise to use functions that account for them.
Returns:

The trimmed data array
Return type:

xr.DataArray
